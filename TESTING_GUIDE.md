## 🧪 快速测试步骤

### 1. 启动应用

```bash
# 如果还没启动，运行：
npm start
```

应用将在 `http://localhost:3000` 运行

### 2. 导航到面试练习

1. 打开应用后登录
2. 点击底部导航的 **"漫学"** 标签
3. 选择 **"面试练习"** 子标签

### 3. 上传简历

在"上传简历"卡片中，选择以下任一方式：

**方式 A：上传文件**
- 点击上传区域
- 选择一个 `.txt`, `.doc`, `.docx`, 或 `.pdf` 文件
- 系统会提示"已选择简历"

**方式 B：手动输入**
- 在文本框中粘贴简历内容
- 包括教育背景、工作经验、技能等信息

### 4. 启动 AI 面试

1. 在"我的面试日程"中选择一个要参加的面试
   - 清华大学 - 建筑学 - 夏令营
   - 同济大学 - 建筑与城规 - 笔试面试
   - 东南大学 - 城乡规划 - 夏令营

2. 点击 **"启动AI面试"** 按钮

3. **第一次调用时可能需要等待 1-2 秒**（豆包 API 首次建立连接）

### 5. 进行面试对话

1. 系统会生成欢迎词和第一个问题
2. 在聊天框输入你的回答
3. AI 将立即给出反馈和下一个问题
4. 重复此过程进行多轮对话

### 6. 结束面试并查看评分

1. 点击聊天页面的"结束面试"或返回按钮
2. 系统会生成总体评分和反馈
3. 查看各项能力评估：
   - 知识掌握
   - 表达能力
   - 学习热情
   - 准备程度

## 🔍 观察和验证

测试时应检查以下内容：

### ✅ 应该看到的

- [ ] 豆包 AI 生成真实、自然的欢迎语
- [ ] 提问内容与简历和学校相关
- [ ] 每个回答后 AI 都有具体的反馈
- [ ] 问题逐渐深入（从自我介绍 → 专业知识 → 研究兴趣）
- [ ] 最后得到数字评分和详细反馈

### ⚠️ 可能遇到的情况

| 情况 | 解决方案 |
|------|--------|
| **API 超时（>5秒）** | 检查网络连接，豆包 API 可能响应慢 |
| **"降级到模拟数据"** | 说明豆包 API 返回错误，已自动使用备用方案 |
| **回答为空** | 检查浏览器控制台是否有错误信息（F12 → Console） |
| **会话不保留** | 这是正常的，刷新页面会清除会话历史 |

## 🐛 调试技巧

### 在浏览器中查看详细日志

1. 按 **F12** 打开开发者工具
2. 选择 **Console** 标签
3. 查看所有 API 调用日志

**检查内容：**
```javascript
// 应该看到类似的日志：
🎤 初始化面试配置和API请求
💬 提交回答到豆包API
✅ 面试评分数据
```

### 查看网络请求

1. 打开开发者工具 → **Network** 标签
2. 启动一个面试
3. 查看是否有请求发送到 `ark.cn-beijing.volces.com`

**预期：**
- 请求 URL: `https://ark.cn-beijing.volces.com/api/v3/chat/completions`
- 请求方法: POST
- 状态码: 200

### 环境变量验证

检查 API Key 是否被正确加载：

在浏览器 Console 中运行：
```javascript
// 检查是否有错误信息
localStorage.getItem('interview_sessions')
```

## 📈 性能指标

目标：
- **首个问题生成:** 1-2 秒
- **回答处理:** 2-3 秒
- **最终评分:** 2-3 秒

如果超过 5 秒，可能是：
1. 网络连接问题
2. 豆包 API 负载高
3. 简历内容过长（>5000 字符）

## 📞 故障排查清单

### 问题：应用无法启动

```bash
# 解决：清理 node_modules 和缓存
rm -r node_modules
npm cache clean --force
npm install
npm start
```

### 问题：OpenAI SDK 导入错误

```bash
# 确保已安装 OpenAI 包
npm list openai
# 如果没有，安装它
npm install openai
```

### 问题：API Key 不生效

```bash
# 1. 检查 .env.local 文件
cat .env.local

# 2. 确保 .env.local 正确放在项目根目录
ls -la | grep env

# 3. 重启开发服务器
npm start
```

### 问题：CORS 错误

由于豆包 API 已配置 CORS，通常不会有问题。如果仍有错误：

```javascript
// 这是预期的配置，保持不变
const client = new OpenAI({
  apiKey: this.ARK_API_KEY,
  baseURL: this.ARK_BASE_URL,
  dangerouslyAllowBrowser: true  // ← 这行是关键
});
```
