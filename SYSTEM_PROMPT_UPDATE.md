# 漫旅面试AI系统提示词更新说明

## 📝 更新内容概览

已成功在面试AI系统中集成了完整的**保研夏令营/预推免面试官**人设和行为规范。

## ✅ 更新详情

### 1. **MockInterviewService.js - 系统提示词升级**

#### 核心变化
```
原系统提示词：简单的通用面试官设定（~150字）
↓
新系统提示词：结构化的漫旅专属规范（~1500字）
```

#### 新系统提示词的六大模块

| 模块 | 内容 | 作用 |
|------|------|------|
| **一、核心身份与规则** | 5条底线规范 | 确保AI不偏离保研面试主题 |
| **二、标准面试流程** | 6个顺序环节 | 引导完整的保研面试流程 |
| **三、提问深度要求** | 4个维度的提问标准 | 保证问题质量和专业度 |
| **四、复盘报告规范** | 结构化输出格式 | 确保评分客观量化 |
| **五、每轮对话格式** | 标准回复模板 | 保证对话清晰有序 |
| **六、必遵守的底线** | 绿灯/红灯清单 | 明确AI的行为边界 |

### 3. **LearnPage.js - 面试城市信息传递**

#### 更新参数
```javascript
// 初始化面试时新增城市参数
initializeInterview({
  resumeText: resumeText,
  schoolName: interview.school,     // 目标院校
  majorName: interview.major,       // 目标专业
  interviewCity: interview.city,    // 新增：面试城市 ✨
  interviewType: interview.type,    // 面试类型
  difficulty: 'medium'
})
```

#### 面试城市的作用
系统提示词现在使用城市信息来：
- 结合城市文化和产业背景出题
- 示例：北京→引入央国企、高新区背景；上海→国际化背景；南京→历史文化底蕴
- 帮学生了解面试所在城市的相关领域特色

## 🎯 系统提示词的核心规范

### 面试流程（严格遵循）
```
1. 自我介绍 (1分钟)
      ↓
2. 简历深挖 (重点环节)
   - 科研项目 → 技术深度
   - 竞赛经验 → 解决问题
   - 课程设计 → 实践应用
      ↓
3. 专业基础
   - 院校特色考点
   - 不出偏题难题
      ↓
4. 院校+城市结合
   - 目标院校研究方向
   - 当地产业/文化背景
      ↓
5. 场景题
   - 科研困难如何解决
   - 研究方向计划
   - 读研规划
      ↓
6. 结束：生成复盘报告
```

### 语气与风格要求
- ✅ 专业、温和、鼓励式
- ✅ 聚焦保研面试场景
- ✅ 给予及时反馈
- ❌ 不闲聊、不跑题、不压力面
