## 🧪 实际操作流程

### 第一步：打开应用
```
http://localhost:3000
```

### 第二步：导航到"通知"页面
```
点击底部导航栏的【通知】图标
或
点击"漫旅 APP"左下角的【通知】
```

### 第三步：查看"邮件解析"标签
```
进入通知页面后，会看到两个标签：
  - 📨 邮件解析  ← 当前已打开
  - 📊 状态漏斗  ← 另一个标签
```

### 第四步：观察邮件智能解析卡片
```
在邮件卡片列表的最上方，会看到：

┌──────────────────────────────────────────┐
│ 邮箱智能解析              ↻ 更新         │
├──────────────────────────────────────────┤
│ ✓  本轮解析完成                          │
│    最后更新：09:24:35                    │
└──────────────────────────────────────────┘

这时处于"完成状态"
```

### 第五步：点击"更新"按钮
```
1. 点击卡片上的【↻ 更新】按钮
2. 按钮立即变为"↻ 解析中..." 并禁用
3. 进度条开始填充
4. 进度点逐个激活
5. 邮件列表变暗（无法点击）
```

### 第六步：观察解析过程

## 💻 交互细节

### 按钮状态变化

#### 解析中（禁用）
```
┌─────────────────┐
│ ↻ 解析中...    │  ← 灰色，无法点击
│ opacity: 0.6    │     光标：not-allowed
│ cursor: disabled│
└─────────────────┘
```

**鼠标状态**：禁用光标（🚫）

#### 解析完成（启用）
```
┌─────────────────┐
│ ↻ 更新 ✨       │  ← 青绿色，悬停高亮
│ Hover时提升     │     光标：pointer
│ transform: y-1px│     阴影：0 4px 12px
└─────────────────┘
```

**鼠标状态**：手指指针（👆）

### 邮件列表状态变化

#### 解析中
```
邮件卡片：
- 背景不透明度：opacity: 0.6（变暗40%）
- 鼠标光标：not-allowed（禁用光标）
- 无法点击
- 提示语：正在智能解析中，请稍候...

动画：进度条、进度点持续动画
    邮件卡片抖动（可选）
```

#### 解析完成
```
邮件卡片：
- 背景不透明度：opacity: 1（恢复）
- 鼠标光标：pointer（手指光标）
- 可点击
- 提示语：漫旅已自动解析你邮箱中的院校...

动画：停止所有动画
```

## 🎯 动画效果详解

### 1. 进度条动画

**效果**：填充 + 微光扫过

```
第一步：条形填充
████████░░░░░░░░░░░ (这个████部分从左向右增长)

第二步：微光闪烁（在████部分）
════════  ← 白色光束从左向右
████████░░░░░░░░░░░

持续周期：2秒
```

**CSS**：
- `animation: progressShimmer 2s infinite`
- 背景渐变：`linear-gradient(90deg, var(--teal), #2aaa94, var(--teal))`

### 2. 进度点动画

```
完成的点 (●)：
  - 颜色：var(--teal) - 青绿色
  - 阴影：0 0 4px rgba(42,122,106,0.4)
  - 无动画

当前点 (◐)：
  - 颜色：var(--teal)
  - 动画：缩放 + 透明度颤动
  - 周期：1.2秒
    0%:   scale(1)    opacity(1.0)
    50%:  scale(1.2)  opacity(0.7)  ← 脉动
    100%: scale(1)    opacity(1.0)
  - 阴影：0 0 6px rgba(42,122,106,0.6)

待解析点 (○)：
  - 颜色：rgba(42,122,106,0.1) - 浅灰
  - 无动画
```

### 3. 完成图标动画

```
出现效果（check pop）：
  0%:   scale(0.8)  opacity(0) ← 从小到大
  100%: scale(1)    opacity(1)
  
  时长：0.4秒
  缓动：cubic-bezier(0.34, 1.56, 0.64, 1) ← 弹性缓出
  
结果：✓ 从小的半透明状态弹出并完全显示
```

### 4. 旋转动画（加载中）

```
刷新按钮上的SVG：
  animation: spin 1.2s linear infinite
  
  0%:   rotate(0deg)
  100%: rotate(360deg)
  
  时长：1.2秒
  缓动：线性（匀速旋转）
```

## 🔊 无障碍设计（a11y）

### 键盘导航
```
Tab键：按钮 → 邮件卡片 → 其他元素
快捷键：
  - Alt+U：刷新邮件（Update）
  - Enter：打开第一封邮件
```

### 屏幕阅读器支持
```
ARIA标签：
  - "邮箱智能解析卡片，2/4邮件已解析"
  - "更新按钮，解析中..."
  - "完成，最后更新时间 09:24"

宣读顺序：
  1. 卡片标题：邮箱智能解析
  2. 状态：工作中/完成
  3. 进度或时间戳
  4. 邮件列表
```

### 颜色对比度
```
- 文字：深灰 on 浅色背景 - WCAG AAA (7:1)
- 按钮：青绿 on 白色背景 - WCAG AA (5.5:1)
- 进度条：青绿 on 浅灰 - WCAG AA (5:1)
```

## 🐛 故障场景处理

### 解析失败
```
预计：显示"解析失败，05秒后自动重试..."
按钮状态：显示重试选项
邮件列表：保持禁用（直到重试成功）
```

### 网络超时
```
预计：显示"网络超时，请检查连接"
按钮状态：显示"重新尝试"
邮件列表：启用（使用缓存数据）
```

### 无邮件解析
```
预计：显示"暂无新邮件需要解析"
按钮状态："更新"可用
邮件列表：为空或显示历史邮件
```
